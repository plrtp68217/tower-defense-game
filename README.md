## Структура проекта

### **СКРИПТЫ**

#### **PlacementSystem** - основная логика строительства
- Использует InputManager для получения пользовательского ввода
- Использует PreviewSystem для отображения превью объекта
- Логика строительства объектов на игровом поле

#### **InputManager** - управление вводом
- Передача координат указателя мыши на выбранной layerMask
- Вызов событий нажатия клавиш мыши и клавиатуры

#### **PreviewSystem** - система предпросмотра
- Логика отображения превью объекта перед строительством

### **КЛАССЫ**

#### **GridData** - управление данными сетки
- Содержит все координаты объектов на игровом поле
- Предоставляет интерфейс добавления новых объектов
- Предоставляет интерфейс проверки возможности размещения объектов

### **БАЗА ДАННЫХ (ScriptableObject)**

#### **ObjectsDatabaseSO** - база данных объектов
- Хранит объекты для размещения на игровом поле

**Структура объекта:**
- **ИМЯ** - название объекта
- **ИДЕНТИФИКАТОР** - уникальный ID
- **РАЗМЕР** (Vector2Int) - габариты объекта
- **ПРЕФАБ** - визуальное представление
